# CashOrderDialog — детальное описание

**Назначение:** Диалог создания и редактирования кассовых ордеров (ПКО — приходный, РКО — расходный) в WPF-приложении BIS.ERP.

## 1. Конструкторы и режимы работы

Диалог имеет два режима:
- **Добавление** — вызывается через конструктор `CashOrderDialog(document, metadataService, orderKind)`, где `orderKind` = `"Receipt"` или `"Payment"`. Заголовок: «Добавление: Приходный/Расходный КО».
- **Редактирование** — вызывается через конструктор `CashOrderDialog(document, metadataService, editId)`. Заголовок: «Редактирование: кассовый ордер».

При создании в режиме добавления автоматически генерируется следующий номер документа через `_metadataService.GetNextCashOrderDocumentNumberAsync`.

## 2. Инициализация данных (`InitializeAsync`)

При открытии окна асинхронно загружаются:
- Справочник касс (`Кассы`) → `CashDeskCombo`
- Организации → `OrganizationCombo`
- Валюты → `CurrencyCombo`
- Материалы → `MaterialCombo`
- Сотрудники → `EmployeeNameBox` (внутренний список)
- Аналитика счетов (`AccountAnalyticsRegistry`)

При редактировании существующего документа (`editId`) additionally заполняются:
- Номер, дата, сумма, основание, примечание
- Выбранная касса, корреспондирующий счет, организация, валюта, сотрудник, материал

## 3. Динамические блоки UI

Видимость полей **Организация**, **Валюта**, **Сотрудник**, **Материал** управляется правилами аналитики счетов (`AccountAnalyticsRules.ShouldShowField`). При смене корреспондирующего счета вызывается `UpdateAccountControlledFieldsVisibility()`, который показывает/скрывает нужные панели в зависимости от настроек выбранного счета.

## 4. Работа с кассой

- **Выбор** — через `ReferenceSelectionDialog` или прямой ComboBox.
- **Добавление** — открывает `CatalogItemDialog`, создает запись в справочнике касс, затем обновляет список.
- **Редактирование** — открывает `CatalogItemDialog` с существующими данными, обновляет запись.
- При выборе кассы в `CashDeskAccountBox` отображается ее счет, а `RefreshPostingPreview()` пересчитывает预览 проводки.

## 5. Работа с корреспондирующим счетом

При нажатии «?» открывается `AccountSelectionDialog` с фильтрацией по модулю документа. Выбранный счет сохраняется в `_selectedCorrAccountId` и `_selectedCorrAccountCode`.

## 6. Работа с сотрудником

Аналогично кассе: выбор через `ReferenceSelectionDialog`, добавление/редактирование через `CatalogItemDialog`. Отображается табельный номер и ФИО.

## 7. Предпросмотр проводки (`RefreshPostingPreview`)

В нижней части диалога отображается группа «Проводка документа» с DataGrid:
- Для ПКО: дебет = счет кассы, кредит = корреспондирующий счет.
- Для РКО: дебет = корреспондирующий счет, кредит = счет кассы.
- Подсказка меняется в зависимости от вида ордера.

При изменении суммы, основания, примечания, кассы или корреспондирующего счета preview автоматически обновляется.

## 8. Сохранение (`OnSaveClick`)

Валидация:
- Номер только цифры.
- Сумма > 0.
- Все видимые поля заполнены (организация, сотрудник, валюта, материал).
- Касса выбрана и имеет заполненный счет.
- Корреспондирующий счет выбран.
- Дебет и кредит не совпадают.

Формируется `itemData` (словарь) с полями:
- `Номер`, `Дата`, `Тип КО`, `Сумма`, `Основание`, `Примечание`, `Проведён` = false
- `Касса`, `Корр. счет`, `Дебет`, `Кредит`, `Сумма в валюте`
- Опционально: `Организация`, `Валюта`, `Сотрудник`, `Материал`

Сохранение:
- Редактирование: `UpdateDynamicRecordAsync`
- Создание: `CreateDynamicRecordAsync`

При успехе `DialogResult = true`, окно закрывается.

## 9. Вспомогательные классы

- `DialogData` — контейнер для данных, загружаемых из БД.
- `CashPostingPreviewRow` — модель строки для preview проводки.
- `CashDeskItem` — расширение `ReferenceItem` для касс с полями `AccountCode`, `CashNumber`, `CurrencyName` и свойством `DisplayNameWithAccount`.

## 10. Особенности

- Ведется отдельный учет номеров для ПКО и РКО, хотя записи хранятся в одной таблице.
- Номера нормализуются через `NormalizeLegacyDocumentNumber` для поддержки старого формата.
- Все операции с БД выполняются асинхронно, UI отображает `WaitCursor`.
- Обработка ошибок через `try/catch` с показом `MessageBox`.
