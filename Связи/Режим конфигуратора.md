# Режим конфигуратора BIS.ERP — подробное описание

## Назначение

**Режим конфигуратора** — это административный интерфейс для управления структурой информационной базы, метаданными, отчетами и печатными формами. Доступ разрешен только администратору (`AuthService.IsAdmin`).

## Основные возможности

### 1. Управление метаданными

#### Справочники (📚)
- Просмотр списка всех справочников
- Редактирование: название, описание, иконка, порядок
- Управление полями: имя, колонка БД, тип (String/Int/Decimal/DateTime/Bool/Reference), длина, обязательность, уникальность
- Настройка ссылок на другие справочники (ReferenceCatalog)
- Шаблоны отображения (DisplayPattern, DisplayFields)
- Расчеты: Depreciation, Sum, Average, Formula
- Правила проводок: дебет/кредит/сумма/условие

#### Документы (📄)
- Аналогично справочникам
- Дополнительно: флаги использования проводок, балансов, движений
- Импорт документов из FoxPro DBF

#### Отчеты и печатные формы (📊)
- Создание новых отчетов через `ReportDesignerWindow`
- Импорт FRX-макетов из FoxPro
- Включение/отключение доступности
- Удаление
- Предпросмотр PDF

### 2. Модули и разделы
- Управление составом документов и отчетов рабочего интерфейса
- Модульная структура системы

### 3. Настройка бухгалтерского учета
- Входящие остатки
- Строки отчетности
- Расчет курсовых разниц

### 4. Пользователи и права
- Управление пользователями информационной базы
- Назначение доступов к разделам рабочего окна
- Интеграция с системой авторизации

### 5. Шаблоны регламентированных отчетов (🧾)
- Загрузка Excel-шаблонов в БД
- Активация шаблонов для автоматического использования
- Редактирование реквизитов (код, название, версия)
- Выгрузка копий шаблонов
- Удаление

### 6. Переводы интерфейса (🌐)
- Единый словарь локализации
- Редактирование переводов по культурам, категориям и ключам
- Активация/деактивация записей

### 7. Импорт/Экспорт

#### Конфигурация
- **Выгрузка**: `.bisconfig` (зашифрованный) или `.json` (открытый)
- **Загрузка**: заменяет текущую структуру конфигурации, отчеты и данные динамических таблиц

#### DBF импорт
- Импорт данных из FoxPro DBF-файлов
- Создание динамических документов

#### Анализ FoxPro
- Анализ структуры FoxPro проекта
- Извлечение метаданных

#### Патчи инфобазы
- Управление миграциями БД
- Применение патчей схемы

### 8. Динамическое создание объектов

#### Динамические справочники
- Создание без программирования
- Автоматическое создание таблицы в БД
- Настраиваемые поля

#### Динамические документы
- Создание с опциональными проводками, балансами
- Автоматическое создание таблицы строк

## Интерфейс

### Структура окна
```
┌─────────────────────────────────────────────────────────────┐
│ Меню: Файл | Создать | Правка | Удалить отчет | Справка    │
├──────────┬──────────────────────────────────────────────────┤
│ Левая    │  Правая панель                                   │
│ панель   │  ┌────────────────────────────────────────────┐ │
│          │  │ Заголовок редактора                         │ │
│ TreeView │  ├────────────────────────────────────────────┤ │
│ с        │  │                                            │ │
│ мета-    │  │  Область свойств / редактор                │ │
│ данными  │  │                                            │ │
│          │  │                                            │ │
│ Кнопки   │  │                                            │ │
│ действий │  └────────────────────────────────────────────┘ │
└──────────┴──────────────────────────────────────────────────┘
```

### Левая панель
- **Дерево метаданных**: Метаданные → Справочники, Документы, Отчеты/Печатные формы, Модули, Настройка учета, Пользователи, Шаблоны отчетов, Переводы
- **Кнопки быстрого действия**: Создать справочник, модули, настройка учета, пользователи, отчет, импорт FRX, шаблоны, выгрузка/загрузка конфигурации, патчи, обновления, режим просмотра

### Правая панель
- Динамически заполняется в зависимости от выбранного объекта
- Для метаданных: обзор с категориями или списком
- Для отдельного объекта: редактор с вкладками (Основные, Поля, Расчеты, Проводки)
- Для отчетов: кнопка открытия в конструкторе, переключатель доступности, удаление, предпросмотр

## Инструкция по взаимодействию

### 1. Создание нового справочника
1. **Создать** → **📚 Справочник** (или кнопка на левой панели)
2. Ввести наименование, описание, иконку
3. После создания открывается редактор:
   - **Основные**: название, описание, иконка, порядок, флаги (проводки/балансы/движения)
   - **Поля**: добавление/удаление полей, настройка типов, ссылок, уникальности
   - **Расчеты**: добавление автоматических расчетов
   - **Проводки**: правила бухгалтерских проводок

### 2. Создание документа
1. **Создать** → **📄 Документ**
2. Аналогично справочнику, но с дополнительными опциями (проводки, балансы)

### 3. Создание отчета
1. **Создать** → **📊 Отчет** или из списка отчетов → **➕ Создать отчет**
2. Открывается `ReportDesignerWindow`:
   - Выбор источника данных (справочник/документ)
   - Добавление полей
   - Настройка фильтров
   - Загрузка FRX-шаблона (опционально)
   - Настройка оформления (шрифты, ориентация, цвета)
   - Сохранение

### 4. Импорт FRX
1. **📁 Импорт FRX** на левой панели
2. Выбрать `.frx` файл
3. Система автоматически парсит макет и создает отчет с типом `FoxProLayout`
4. После импорта можно открыть в конструкторе для редактирования

### 5. Управление отчетами
- Из дерева метаданных или списка отчетов:
  - **Открыть в конструкторе** — полное редактирование
  - **Сделать доступным/отключить** — управление видимостью в рабочем режиме
  - **Предпросмотр** — быстрый PDF без открытия конструктора
  - **Удалить** — с подтверждением

### 6. Работа с шаблонами регламентированных отчетов
1. **🧾 Шаблоны отчетов**
2. **Загрузить шаблон** — выбрать Excel файл, заполнить реквизиты (код, название, версия)
3. **Сделать активным** — один шаблон на код может быть активен
4. **Изменить реквизиты** — редактирование метаданных без перезагрузки файла
5. **Выгрузить копию** — скачать шаблон из БД

### 7. Импорт/Экспорт конфигурации
- **Выгрузить**: сохраняет всю конфигурацию (метаданные, отчеты, данные) в файл
  - `.bisconfig` — зашифрованный формат
  - `.json` — открытый формат
- **Загрузить**: заменяет текущую конфигурацию (с подтверждением)

### 8. Локализация
- **🌐 Переводы интерфейса**
- Таблица: Язык | Ключ | Перевод | Категория | Активен
- Возможность добавления, редактирования, удаления записей
- Изменения применяются immediately через `LocalizationService`

### 9. Патчи и обновления
- **🧩 Патчи инфобазы** — применение SQL-миграций к текущей БД
- **🚀 Обновления программы** — управление версиями приложения

## Ключевые особенности

- **Админский доступ**: проверяется при загрузке конфигуратора
- **Динамическая загрузка**: все элементы интерфейса создаются программно в зависимости от данных
- **Интеграция с метаданными**: центральное хранилище `MetadataObject` с поддержкой полей, расчетов и проводок
- **FoxPro-наследие**: поддержка импорта FRX/DBF для миграции со старых систем
- **Расширяемость**: возможность создания новых справочников и документов без изменения кода

## Пример workflows

### Миграция отчета из FoxPro
1. **Создать** → **📊 Отчет** (пустой)
2. В конструкторе: **📂 Загрузить FRX-файл**
3. Система парсит FRX, извлекает поля и макет
4. Настроить соответствия полей (вкладка "FRX поля")
5. Сохранить отчет
6. В конфигураторе: правый клик на отчет → **Сделать доступным**

### Создание нового справочника "Контрагенты"
1. **Создать** → **📚 Справочник**
2. Заполнить основные данные
3. Добавить поля: Наименование, ИНН, Адрес, Телефон
4. Настроить отображение (DisplayPattern)
5. Добавить расчет суммы если нужно
6. Настроить проводки (если используется в документах)
7. Сохранить
8. Справочник автоматически появляется в рабочем режиме
