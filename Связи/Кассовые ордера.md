# CashOrderWorkView — модуль кассовых ордеров

## Назначение

`CashOrderWorkView` — это `UserControl` для полноценной работы с приходными и расходными кассовыми ордерами (ПКО/РКО) в рабочем интерфейсе BIS.ERP. Предоставляет:
- Просмотр списка всех кассовых ордеров
- Создание новых (ПКО/РКО)
- Редактирование непроведенных документов
- Проведение/отмена проведения
- Печать через выбор печатной формы
- Удаление
- Просмотр проводки по двойному клику

## Виды кассовых ордеров

Система поддерживает **два вида**, которые хранятся в одной таблице `doc_cash_orders`:

### 1. Приходный кассовый ордер (ПКО) — Receipt
- Тип проводки: **Дебет** счета кассы / **Кредит** корреспондирующего счета
- Отображается как `"Приходный"` в колонке `Тип КО`
- Кнопка создания: `➕ Приходный КО`
- При печати: ищет формы с префиксом `cash.receipt.`

### 2. Расходный кассовый ордер (РКО) — Payment
- Тип проводки: **Дебет** корреспондирующего счета / **Кредит** счета кассы
- Отображается как `"Расходный"` в колонке `Тип КО`
- Кнопка создания: `➖ Расходный КО`
- При печати: ищет формы с префиксом `cash.payment.`

**Определение типа** при загрузке из БД (`ResolveOrderKind`):
1. Проверяет поле `Тип КО`, `order_kind`, `cash_order_kind`, `Тип`, `document_type`
2. Если содержит "приход" или равно "Receipt" — ПКО
3. Иначе — РКО

## Единая таблица хранения

Оба типа ордеров хранятся в одной таблице **`doc_cash_orders`**. Поля таблицы:

| Поле | Назначение |
|------|------------|
| `Id` | UUID |
| `Номер` (doc_number) | Номер документа |
| `Дата` (doc_date) | Дата документа |
| `Тип КО` (order_kind) | Receipt / Payment |
| `Сумма` (amount) | Сумма в сомах |
| `Сумма в валюте` (amount_currency) | Сумма в валюте |
| `Основание` (basis) | Основание операции |
| `Примечание` (description) | Примечание |
| `Проведён` (is_posted) | Флаг проведения |
| `Касса` (cash_desk_id) | Ссылка на справочник "Кассы" |
| `Корр. счет` (correspondent_account) | Корреспондирующий счет |

## Нумерация и счетчик

### Независимая нумерация
- Кассовые ордера имеют **независимую нумерацию** от других документов
- Определяется в `MetadataService.IndependentDocumentNumberingNames` = `{ "Расходный/Приходный КО" }`
- Следующий номер генерируется через `_metadataService.GetNextCashOrderDocumentNumberAsync` (см. CashOrderDialog)

### Номер документа
- Номер генерируется автоматически при создании нового КО
- Можно **редактировать** в диалоге: в поле "Номер" есть чекбокс **"изменить"**, который делает поле редактируемым
- При редактировании существующего документа номер загружается из базы

## Зависимость счетов

### Логика построения проводки (`CashOrderDialog.BuildPostingAccounts`)

```csharp
private (string DebitAccount, string CreditAccount) BuildPostingAccounts(string cashDeskCode, string corrAccountCode)
    => IsReceiptOrder(_orderKind)
        ? (cashDeskCode, corrAccountCode)    // ПКО: Дт касса / Кт корр.счет
        : (corrAccountCode, cashDeskCode);    // РКО: Дт корр.счет / Кт касса
```

### Выбор кассы
- Касса выбирается из справочника "Кассы"
- При выборе кассы подставляется ее счет (поле `Счет` = код счета из Плана счетов)
- Можно: выбрать "?", добавить "+", редактировать "Изм." кассу

### Выбор корреспондирующего счета
- Выбирается через `AccountSelectionDialog`
- Фильтруется по модулю документа
- Можно задать вручную или выбрать из Плана счетов

### Аналитика счетов
- При выборе корреспондирующего счета система проверяет `AccountAnalyticsRegistry`
- В зависимости от настроек счета показываются/скрываются поля: **Организация**, **Валюта**, **Сотрудник**, **Материал**

### Валидация при сохранении
- Номер должен содержать только цифры
- Сумма > 0
- Все видимые поля (зависимые от аналитики) заполнены
- Касса выбрана и имеет счет
- Корреспондирующий счет выбран
- Дебет и кредит не совпадают

## Возможность редактирования номеров документов

- **Новые документы**: номер генерируется автоматически, но перед сохранением можно включить чекбокс "изменить" и ввести свой номер
- **Редактирование**: при открытии существующего документа на редактирование номер подставляется из БД, но его также можно изменить через чекбокс
- **Редактирование запрещено**, если документ проведен (кнопка `✏️ Редактировать` отключена)

## Проведение документа

### Процесс проведения (`PostDocumentAsync`)
1. Проверка дебета ≠ кредита
2. Создание/обновление записи в `doc_postings` с проводкой
3. Установка `is_posted = true`

### Отмена проведения (`UnpostDocumentAsync`)
1. Удаление проводки из `doc_postings`
2. Установка `is_posted = false`

### Визуальные индикаторы
- Проведенные строки подсвечиваются **зеленым** фоном (`LightGreen`)
- Кнопка проведения меняет текст:
  - Для непроведенного: `✅ Провести` (ширина 100px)
  - Для проведенного: `↩ Отменить проведение` (ширина 175px)
- Колонка "Проведён" = "Да" / "Нет" (через `LocalizationService.DisplayValue`)

## Печать

При нажатии `🖨 Печатная форма`:
1. Определяется тип КО (ПКО/РКО) → ищутся формы с префиксом `cash.receipt.` или `cash.payment.`
2. Открывается `PrintFormSelectionDialog` для выбора конкретной формы
3. Формируется PDF через `PrintFormService.ExportDocumentAsync`
4. Открывается `PdfPreviewWindow` для предпросмотра

## Модель `CashOrderRow`

```csharp
public class CashOrderRow {
    public Guid Id { get; set; }
    public Guid DocumentMetadataId { get; set; }
    public MetadataObject? DocumentMetadata { get; set; }
    public string DocumentType { get; set; }    // "Расходный/Приходный КО"
    public string OrderKind { get; set; }       // "Payment" / "Receipt"
    public bool IsReceipt => OrderKind == "Receipt";
    public string OrderTypeDisplay { get; }     // "Приходный" / "Расходный"
    public string PostingDocumentType { get; }  // "Приходный кассовый ордер" / "Расходный кассовый ордер"
    public string PostingTypeDisplay { get; }   // "Приход: Дт касса / Кт корр. счет" / "Расход: Дт корр. счет / Кт касса"
    public string DocNumber { get; set; }
    public DateTime DocDate { get; set; }
    public string CashDeskName { get; set; }
    public string OrganizationName { get; set; }
    public string CurrencyName { get; set; }
    public string EmployeeName { get; set; }
    public string MaterialName { get; set; }
    public decimal Amount { get; set; }
    public string Basis { get; set; }
    public string CorrespondentAccountName { get; set; }
    public string Description { get; set; }
    public bool IsPosted { get; set; }
    public string IsPostedDisplay { get; }
    public DateTime CreatedAt { get; set; }
    public DateTime UpdatedAt { get; set; }
    public string DebitAccount { get; set; }
    public string CreditAccount { get; set; }
    public decimal AmountInCurrency { get; set; }
    public string CashDeskId { get; set; }
}
```
