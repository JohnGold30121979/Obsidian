# ReportDesignerWindow — описание модуля

**Назначение:** Визуальный конструктор отчетов для создания, редактирования и предпросмотра бухгалтерских и управленческих отчетов в BIS.ERP.

## Архитектура UI
Максимизированное окно с верхней панелью инструментов, `TabControl` с 5 вкладками и нижней статусной панелью.

## Основные вкладки

### 1. 📋 Общие
Основные параметры отчета:
- **Название, описание**
- **Источник данных** — выбор справочника/документа из метаданных
- **Тип отчета:** Таблица, Список, Карточка, Счет-фактура на материалы, Расходный/Приходный КО, Макет Visual FoxPro FRX
- **Заголовок, подзаголовок, текст в шапке/подвале, итоговый текст**
- **FRX-шаблон** — загрузка `.frx` файла с парсингом в JSON
- Флаги: доступен, печатная форма документа, по умолчанию

### 2. 📊 Поля [[вкладку FRX поля]]
Управление полями отчета:
- Левая панель: доступные поля из выбранного источника данных
- Правая панель: таблица полей отчета с возможностью:
  - Добавление/удаление полей (двойной клик или кнопки)
  - Изменение: отображаемое имя, тип агрегации (Sum, Average, Count, Min, Max), ширина, видимость, выравнивание
  - Переупорядочивание (⬆ Вверх / ⬇ Вниз)

### 3. 🔗 FRX поля
Работа с Legacy FoxPro макетами:
- Таблица соответствий элементов FRX-макета полям данных
- Для каждого элемента: тип, положение, поле данных, формат, произвольный текст, видимость
- Предпросмотр соответствий в реальном времени
- Активируется только после загрузки FRX-файла

### 4. 🧱 Нативный макет
Визуальный дизайнер для создания отчетов с нуля:
- **Три колонки:** элементы макета | холст Canvas | свойства элемента
- **Банды:** Header, Body, Footer
- **Элементы:** Text, Expression, Line, Box
- Возможность точного позиционирования (X, Y, Width, Height) и редактирования свойств (шрифт, размер, выравнивание, bold/italic)
- Масштабирование от 4% до 400%
- Создание нового пустого макета

### 5. 🎨 Оформление
Настройки внешнего вида:
- Ориентация страницы: Книжная/Альбомная
- Шрифт: Segoe UI, Arial, Times New Roman, Calibri
- Размер шрифта: 8–18
- Цвет шапки: 5预设 цветов
- Чередование цветов строк, линии сетки, общий итог, номера страниц

### 6. 🔍 Фильтры
Условия отбора данных:
- Динамическое добавление условий
- Поля: выпадающий список доступных полей
- Операторы: =, <, >, <=, >=, Like
- Значение: текстовый ввод

## Ключевая логика

### Загрузка данных
- При инициализации загружает все каталоги и документы через `MetadataService`
- При выборе источника подгружает его поля в `AvailableSourceFields`, `AvailableDataFields`, `AvailableFilterFields`
- Поддерживает computed fields из `PrintFormService`

### Работа с FRX
- Парсинг [[FRX]] через `PrintFormService.ParseFrxFileTemplateAsync`
- Извлечение полей через `PrintFormService.ExtractReportFieldsFromTemplate`
- Десериализация макета и построение соответствий элементов
- Автоматическое переключение типа отчета на "FoxProLayout"
- Отложенная загрузка нативного шаблона (`DeferNativeTemplate`)

### Сохранение
- Создание нового `Report` или обновление существующего
- **Важно:** при редактировании старый отчет удаляется вместе с полями, фильтрами и соответствиями, создается новый с тем же Id и CreatedAt
- Сохраняет: `Report`, `ReportField`, `ReportFilter`, `ReportElementMapping`
- Сохраняет нативный шаблон в `TemplateTextBox.Text`

### Предпросмотр и экспорт
- **Печатная форма:** `_printFormService.ExportTemplatePreview(report)` → `PdfPreviewWindow`
- **Обычный отчет:** `_reportService.GetReportDataAsync` + `_reportService.ExportToPdf`
- Экспорт в файл через `SaveFileDialog`

## Вспомогательные классы
- `FieldDef` — описание поля (Name, DbColumnName, Type)
- `FrXElementMappingViewModel` — модель для соответствия FRX-элементов, реализует `INotifyPropertyChanged`

## Интеграция
- Использует `ServiceLocator.InfoBaseManager.GetCurrentDbContextAsync()` для получения контекста БД
- Взаимодействует с `ReportService`, `PrintFormService`, `MetadataService`
- Данные отображаются через `DataGrid`, `ListBox`, `Canvas` (нативный дизайнер)


В визуальном построителе отчетов добавил общую базу правил FoxPro-полей и редактор правил прямо на вкладке `FRX поля`.
