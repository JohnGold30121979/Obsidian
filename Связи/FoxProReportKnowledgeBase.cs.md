# Правила распознавания FoxPro-полей в FRX

Этот документ описывает, как настраивать общие правила распознавания FoxPro-полей для импортированных FRX-отчетов в визуальном построителе BIS ERP.

Главная идея: один раз связать старое FoxPro-поле или выражение с нормальным полем текущей системы, сохранить это как правило, и потом применять автоматически к похожим отчетам.

## Где настраивать

1. Откройте  Конфигуратор ([[Режим конфигуратора]])
2. Перейдите в раздел отчетов и откройте нужную печатную форму FRX в редакторе.

3. Откройте [[вкладку FRX поля]].

4. В верхней таблице `Соответствие элементов макета полям данных` выберите текущие поля данных для элементов FRX.

5. Ниже находится блок `Общие правила распознавания FoxPro-полей`.

На вкладке есть три основные кнопки:

| Кнопка | Что делает |

| --- | --- |

| `Применить общие правила` | Берет уже сохраненные правила и пытается автоматически заполнить пустые привязки элементов FRX. |

| `Сохранить привязки как правила` | Берет текущие вручную выбранные привязки и превращает их в общие правила для будущих похожих отчетов. |

| `Сохранить список правил` | Сохраняет изменения в нижней таблице правил: ручные добавления, удаления, правки приоритета, regex и активности. |

## Что такое правило

Правило отвечает на вопрос:

> Если в FRX встречается старое поле FoxPro `X`, какое поле текущей системы нужно подставить вместо него?

Правило состоит из таких колонок:

| Колонка | Значение |

| --- | --- |

| `FoxPro поле/выражение` | Старое имя поля или выражение из FRX. Например `name_kod`, `VED.NAME_KOD`, `debsum`, `kod_arm`, `DTOC(DATE)`. |

| `Каноническое поле` | Внутренний смысл поля. Например `operation_name`, `debit_amount`, `module`. |

| `Поле данных` | Реальное поле текущего отчета BIS ERP, которое будет подставляться. Например `operation_name` или `Модуль`. |

| `Профиль` | Код профиля распознавания. Для актов сверки обычно `foxpro_reconciliation_act`. |

| `Приоритет` | Чем меньше число, тем раньше правило применяется. Обычно достаточно `100`. |

| `Regex` | Если включено, `FoxPro поле/выражение` трактуется как регулярное выражение. |

| `Активно` | Если выключить, правило хранится, но не применяется. |

| `Комментарий` | Заметка для себя и команды. |

## База знаний распознавания

Общая база знаний находится в классе:

[FoxProReportKnowledgeBase.cs](E:/C#/BIS.ERP/BIS.ERP/Services/FoxProReportKnowledgeBase.cs)

Она содержит встроенные правила нормализации и алиасы. Это нужно, чтобы не создавать отдельное правило на каждую вариацию одного и того же поля.

### Что база знаний делает автоматически

Она нормализует выражения FoxPro:

| FRX выражение | Нормализуется к |

| --- | --- |

| `{name_kod}` | `name_kod` |

| `=name_kod` | `name_kod` |

| `ALLTRIM(name_kod)` | `name_kod` |

| `DTOC(date)` | `date` |

| `VED.NAME_KOD` | `VED.NAME_KOD`, затем понимает базовое поле `NAME_KOD` |

| `DB_CR.DATOBR` | `DB_CR.DATOBR`, затем понимает базовое поле `DATOBR` |

Она понимает старые префиксы таблиц FoxPro:

| Префикс | Откуда обычно |

| --- | --- |

| `ved`, `ved2`, `ved3`, `ved4` | Ведомости, обороты, промежуточные таблицы |

| `db_cr`, `db_crs`, `dbcr`, `dbcrs` | Данные проводок и остатков |

| `ksprorg` | Справочник организаций |

| `avt_p` | Справочник авансовых платежей / пар счетов |

| `fact`, `curFACTSW`, `irfactsw` | Счет-фактуры |

Она добавляет алиасы для строк отчета:

| Пример алиаса | Для чего нужен |

| --- | --- |

| `line1_name_kod` | Первая строка отчета |

| `line1.name_kod` | Первая строка отчета в стиле FoxPro |

| `ved.name_kod` | Старое поле из VED |

| `ved3_debsum` | Старое поле из VED3 |

| `db_crs.korsch` | Старое поле из DB_CRS |

## Канонические поля

Каноническое поле - это нормальный внутренний смысл старого FoxPro-поля.

### Акт сверки

| Каноническое поле | Старые FoxPro-алиасы | Текущие поля BIS ERP |

| --- | --- | --- |

| `operation_name` | `name_kod`, `naim`, `name`, `operation`, `material_name` | `Наименование материала, вид операции`, `operation_name` |

| `debit_account` | `schet`, `deb`, `debet`, `debit` | `Дебет`, `debit_account` |

| `credit_account` | `kor_sch`, `korsch`, `cred`, `credit` | `Кредит`, `credit_account` |

| `debit_amount` | `debsum`, `sum_deb`, `deb_beg`, `deb_v` | `Сумма Дт`, `debit_amount` |

| `credit_amount` | `credsum`, `sum_cred`, `cred_beg`, `cred_v` | `Сумма Кт`, `credit_amount` |

| `document_number` | `dok`, `dokum`, `nom_dok`, `doc` | `N докум`, `document_number` |
